# cc-gym 产品定义

## 一句话

在 Claude Code 忙着干活的时候，cc-gym 把你从"盯屏发呆"里拽出来，做一个 30 秒的小动作。

## 要解决的问题

和 Claude 协作编程时，真正的空档是 **CC 在跑的那段时间**——你提完需求，工具链在后台执行，你要么盯着输出发呆，要么切去刷别的东西、被心智切换搞得心烦意乱，回来还得重新进入状态。

通用番茄钟/久坐提醒不知道你"正在等 CC 还是正在思考"，容易打断心流；而一轮回合结束后的提醒又晚了——那时候你已经要开始看结果、写下一条指令了，被提醒反而是打扰。

cc-gym 的时机锚点是 **CC 的忙碌时段**：工具一开始跑，你就被温柔地推一个小动作；等你做完，结果也差不多出来了。

## 核心价值

| | cc-gym | 通用久坐提醒 | Stop 钩子式提醒 |
|---|---|---|---|
| 时机 | CC 跑工具时，你正好闲 | 固定间隔，可能正专注时弹 | 回合结束，你正要打字 |
| 匹配感 | 提醒 = 可利用的等待窗口 | 无关业务节奏 | 时机偏晚，像催你 |
| 接入 | 装一次全局生效 | 独立 App | 同 cc-gym |

## 核心机制：忙碌时长感知

cc-gym 不预测哪条命令会长跑，而是**测量实际忙碌时长**。理由：`git log` 可能瞬间完成也可能卡住，`npm test` 可能 2 秒也可能 5 分钟，启发式注定不准。

状态机（按 session 隔离）：

- 你提交 prompt → 记录 `busy_start = now`，清空提醒标记
- 每次工具调用前检查 `elapsed = now - busy_start`：
  - `elapsed ≥ 20s` 且尚未提醒轻档 → 弹 **轻提醒**，置标记
  - `elapsed ≥ 180s` 且尚未提醒重档 → 弹 **重提醒**，置标记
- 权限确认 / 空闲等待弹窗出现 → 记录暂停点；下次事件触发时把这段等待时长从 `elapsed` 里扣除（那段是你在忙，不是 CC）
- CC 回合结束 → 清空所有状态，等下一轮

单轮回合里每档最多弹一次，不会连环骚扰。

## 核心功能

### F1 · 双层阈值提醒（MVP）

- **轻档（20 秒）**：从分类池（眼/颈/呼吸）挑一个 30 秒内能做完的小动作——转脖子、深呼吸、望远、耸肩、手腕转圈
- **重档（3 分钟）**：从分类池（眼/腿）挑一个起身 1-2 分钟的动作

分两档是为了**匹配等待的不确定性**：你不知道 CC 会跑 40 秒还是 5 分钟，所以先给你个 30 秒级动作保底；如果真跑很久，再升级成"认真活动"。

挑选规则是**按类别轮转**——先在"上次之外的类别"里均匀选一类，再在类内均匀选一条。不会连续两次推同类，长期下来眼/颈/腿/呼吸分布均匀，不被条目多寡带偏。

### F2 · 可中断原则

推送的永远是"做到哪算哪"的动作，不是"必须完成 90 秒组合拉伸"。CC 跑完你回来就回来，没做完不心疼——这让被打断的成本接近 0，消解了"前半段空等 + 后半段被催回"的焦虑。

### F3 · 无感冷却

- 每轮回合里每档最多一次，不连环骚扰
- 全局冷却（默认 15 分钟）：不管几个 session 并行，同一冷却窗口里只会弹一次，其他被安静抑制——按"防久坐、不打断专注"的节奏兜底，避免 3 个 CC 同时跑时被三连弹
- 短任务（<20s）天然不触发，不存在"5 秒的 Read 也要提醒我"的骚扰
- 回合结束即重置，下一轮从头计时

### F4 · 连续短命令也算忙

关键场景：单个 Bash 只跑 2 秒，但 CC 连续跑了 30 个。累计起来你已经盯屏 1 分钟了。因为阈值比较的是 **"自 prompt 提交以来的累计时间"**，这种场景照常触发，不会漏。

## 交互流程

**用户视角的完整体验：**

1. 一次性安装 → cc-gym 注入 UserPromptSubmit / PreToolUse / Notification / Stop 钩子
2. 照常写 prompt，不感知 cc-gym 存在
3. CC 开始跑工具，20 秒后系统通知弹出：「转转脖子，各方向各 5 秒」
4. 你做或不做，通知自动消失；CC 还没跑完也不会再打扰这轮
5. 如果 CC 跑超 3 分钟，再弹一次「起身走动 1-2 分钟」
6. CC 回合结束，状态重置；下一条 prompt 进入新周期

**用户不需要：**
- 开 App、登录、看仪表盘
- 手动"开始专注"或"结束专注"
- 回应提醒（没有"完成/跳过"按钮）

## 产品边界

**在范围内：**
- 绑定 CC 忙碌时段的提醒
- 本地运行、本地配置
- 简单清单式的运动内容

**明确不做：**
- **不做统计追踪** — 不记录做没做、做了多少
- **不做云端/社交** — 不同步、不分享、不排行
- **不做健康指导** — 不是健身 App，只是温柔提醒
- **不替代专业软件** — 眼保健/久坐提醒有更专业的产品
- **不做 GUI** — 没有设置界面，配置即环境变量
- **不提醒非 CC 场景** — 不当通用番茄钟用
- **不在 CC 回合结束时提醒** — 那时你正要回到键盘，提醒是打扰

## 配置能力

用户可调项：

| 可调项 | 环境变量 | 典型值 | 作用 |
|---|---|---|---|
| 轻档阈值 | `CC_GYM_LIGHT_SEC` | 20 / 30 / 45 秒 | 多快弹第一次提醒 |
| 重档阈值 | `CC_GYM_HEAVY_SEC` | 180 / 300 秒 | 多久后升级到"认真活动" |
| 全局冷却 | `CC_GYM_GLOBAL_COOLDOWN` | 默认 900 秒（15 分钟） | 跨会话两次弹窗的最短间隔，兜底节奏 |
| 运动清单 | `CC_GYM_LIGHT_FILE` / `CC_GYM_HEAVY_FILE` | 任意文本文件 | 替换内置轻/重档清单 |

运动清单格式：每行 `category: 动作描述`，`category ∈ {eye, neck, leg, breath}`，`#` 开头为注释。

## 衡量标准

这个工具做得好不好，就看两件事：

1. **装上之后还在用** — 没被烦到卸载，说明双档节奏合适
2. **真的活动过** — 哪怕一天只因它动过两三次，就是成功

## 后续可能性（暂不做）

记录在案，等真用一段时间再判断要不要做：

- 根据历史忙碌分布自适应调阈值
- 运动按时段挑（早/晚不同类型）
- 状态行显示"本轮已忙 X 秒"
